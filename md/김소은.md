# 기능 명세서 — 김소은

> 담당: F4 AI 루트 결과 화면 + B3 AI 루트 생성 API

---

## 서비스 소개

꼬잉꼬잉은 지역 기반 당일치기 여행지를 추천해주는 서비스입니다.
"극 P의 여행"을 컨셉으로, 사용자가 직접 찾지 않아도 관련 정보와 추천이 자연스럽게 제공되는 큐레이션 여행 서비스입니다.

## 담당 기능 선정 이유

F4 AI 루트 결과 화면과 B3 AI 루트 생성 API는 꼬잉꼬잉의 핵심 차별점입니다. 단순히 장소를 나열하는 것에서 나아가, 선택한 여행지들을 AI가 최적 동선으로 정렬해 지도 위에 시각화해줍니다. "어디 가지?" 고민을 해결한 뒤 "어떤 순서로 가지?"까지 해결해주는 기능입니다.

## AI 대화 요약

꼬잉꼬잉 서비스 기획 방향 논의부터 시작해서,
전체 화면(F1~F8) 및 API(B1~B7) 구조를 도출하고,
담당 기능인 (본인 기능)의 화면 구성과 API 명세를
AI와의 대화를 통해 구체화하였습니다

---

## F4 — AI 루트 결과 화면

### 개요

| 항목      | 내용                                                                  |
| --------- | --------------------------------------------------------------------- |
| 화면 ID   | F4                                                                    |
| 화면명    | AI 루트 결과 화면                                                     |
| 진입 경로 | F2 컨텐츠 상세 하단 "AI 루트 만들기" CTA / F8 룰렛 결과 "루트 만들기" |
| 연결 API  | B3 (AI 루트 생성)                                                     |

---

### 화면 구성

#### 지도 영역

| 요소          | 설명                                                        |
| ------------- | ----------------------------------------------------------- |
| 지도          | 전체 화면의 상단 60% 차지                                   |
| 루트 시각화   | 선택된 장소들을 순서대로 핀으로 표시, 이동 경로 선으로 연결 |
| 핀 번호       | 방문 순서대로 번호 표시 (①②③…)                              |
| 지도 인터랙션 | 핀치 줌, 드래그 이동 가능                                   |

#### 루트 정보 카드

| 요소              | 설명                       |
| ----------------- | -------------------------- |
| 총 예상 이동 시간 | 출발지~최종 목적지 총 시간 |
| 총 거리           | 전체 이동 거리 (km)        |
| 장소 수           | 루트에 포함된 장소 수      |

#### 장소 목록 (세로 스크롤)

| 요소                   | 설명                                     |
| ---------------------- | ---------------------------------------- |
| 방문 순서 번호         | 좌측에 크게 표시                         |
| 장소 썸네일            | 소형 이미지                              |
| 장소명 + 카테고리 태그 |                                          |
| 이동 시간              | 이전 장소에서 이 장소까지 예상 이동 시간 |
| 장소 탭                | F2 컨텐츠 상세로 이동                    |

#### 하단 액션 버튼

| 버튼        | 동작                                     |
| ----------- | ---------------------------------------- |
| 루트 저장   | 현재 루트를 소장함(F5)에 저장            |
| 공유하기    | 루트 링크 또는 이미지로 공유             |
| 루트 재생성 | 같은 조건으로 다른 루트 생성 (B3 재요청) |

---

### 기능 요구사항

- 진입 시 B3 API에 선택한 장소 목록 전달하여 최적 동선 요청
- 로딩 중 스켈레톤 UI 또는 "AI가 최적 루트를 계산 중이에요..." 메시지 표시
- 지도 핀 탭 시 해당 장소 상세 카드 팝업 표시
- 장소 목록과 지도 핀이 연동 (목록 탭 시 지도 해당 핀 강조)
- 루트 저장은 로그인 사용자만 가능, 비로그인 시 로그인 유도 팝업

---

### 화면 플로우

```
F2 컨텐츠 상세 / F8 룰렛 결과
    ↓ "AI 루트 만들기"
B3 API 요청 (로딩)
    ↓
AI 루트 결과 화면
    ↓ 장소 탭
F2 컨텐츠 상세

    ↓ 루트 저장
F5 소장함
```

## B3 — AI 루트 생성 API

### 개요

| 항목      | 내용                                 |
| --------- | ------------------------------------ |
| API ID    | B3                                   |
| API명     | AI 루트 생성 API                     |
| 연결 화면 | F4 AI 루트 결과 화면, F2 컨텐츠 상세 |

---

### 엔드포인트 목록

#### 1. AI 루트 생성

| 항목   | 내용                                                    |
| ------ | ------------------------------------------------------- |
| Method | POST                                                    |
| URL    | `/api/routes/generate`                                  |
| 설명   | 입력된 장소 목록을 기반으로 최적 동선 계산 후 루트 반환 |

**Request Body**

| 파라미터      | 타입   | 필수 | 설명                                                |
| ------------- | ------ | ---- | --------------------------------------------------- |
| contentIds    | array  | Y    | 루트에 포함할 장소 ID 목록                          |
| startLocation | object | N    | 출발지 좌표 (lat, lng). 미입력 시 첫 번째 장소 기준 |
| travelType    | string | N    | `daytrip` 또는 `overnight` (기본: daytrip)          |

**Response**

| 필드            | 타입   | 설명                                            |
| --------------- | ------ | ----------------------------------------------- |
| routeId         | string | 생성된 루트 ID                                  |
| orderedContents | array  | 최적 순서로 정렬된 장소 목록                    |
| totalDuration   | number | 총 예상 이동 시간 (분)                          |
| totalDistance   | number | 총 이동 거리 (km)                               |
| legs            | array  | 구간별 이동 정보 (from, to, duration, distance) |
| mapPolyline     | string | 지도 경로 폴리라인 데이터                       |

**orderedContents 항목 구조**

| 필드             | 타입   | 설명                         |
| ---------------- | ------ | ---------------------------- |
| order            | number | 방문 순서                    |
| contentId        | string | 장소 ID                      |
| name             | string | 장소명                       |
| latitude         | float  | 위도                         |
| longitude        | float  | 경도                         |
| durationFromPrev | number | 이전 장소에서 이동 시간 (분) |

---

#### 2. 루트 상세 조회

| 항목   | 내용                       |
| ------ | -------------------------- |
| Method | GET                        |
| URL    | `/api/routes/{routeId}`    |
| 설명   | 저장된 루트 상세 정보 조회 |

---

#### 3. 루트 저장

| 항목   | 내용                                         |
| ------ | -------------------------------------------- |
| Method | POST                                         |
| URL    | `/api/routes/{routeId}/save`                 |
| 설명   | 생성된 루트를 사용자 소장함에 저장 (B5 연동) |
| 인증   | 로그인 필요                                  |

---

### 루트 최적화 알고리즘

| 항목         | 설명                                                                                          |
| ------------ | --------------------------------------------------------------------------------------------- |
| 구현 방식    | Kakao Maps / Google Maps Directions API를 활용하여 경유지 최적 순서 계산 (외부 지도 API 사용) |
| 기준         | 총 이동 시간 최소화                                                                           |
| 이동 수단    | 기본: 자동차 기준 (추후 대중교통 옵션 추가 고려)                                              |
| 장소 수 제한 | 최대 5개 (당일치기 기준)                                                                      |

> **구현 방식 참고:** 외부 지도 API(Kakao Maps 또는 Google Maps)의 경유지 최적화 기능을 활용하는 방식으로 구현 예정입니다. 최종 프로젝트 진행 시 API 선택 및 비용 여부 확인 후 확정합니다.

---

### 비기능 요구사항

- 루트 생성 응답 시간 목표: 3초 이내
- 동일 장소 조합에 대한 루트는 캐싱 처리 (1시간)
- 장소 좌표 데이터는 B1 컨텐츠 DB에서 조회
- 외부 지도 API 사용 시 API 키 환경변수 관리
