# 기능 명세서 — 우승연

> 담당: F3 필터/검색 UI + B4 필터·검색 API

---

## 서비스 소개

꼬잉꼬잉은 지역 기반 당일치기 여행지를 추천해주는 서비스입니다.
"극 P의 여행"을 컨셉으로, 사용자가 직접 찾지 않아도 관련 정보와 추천이 자연스럽게 제공되는 큐레이션 여행 서비스입니다.

## 담당 기능 선정 이유

F3 필터/검색 UI는 사용자가 원하는 여행지를 직접 좁혀가는 화면입니다. 지역·테마·여행유형을 자유롭게 조합해 복합 조건으로 검색할 수 있는 것이 꼬잉꼬잉의 핵심 탐색 방식입니다. B4 필터·검색 API는 이 조건을 실제로 처리해 결과를 반환하며, F6 온보딩·F7 테마 탐색 등 다른 화면과도 연결됩니다.

## AI 대화 요약

꼬잉꼬잉 서비스 기획 방향 논의부터 시작해서,
전체 화면(F1~F8) 및 API(B1~B7) 구조를 도출하고,
담당 기능인 (본인 기능)의 화면 구성과 API 명세를
AI와의 대화를 통해 구체화하였습니다

## F3 — 필터 / 검색 UI

### 개요

| 항목      | 내용                             |
| --------- | -------------------------------- |
| 화면 ID   | F3                               |
| 화면명    | 필터 / 검색 UI                   |
| 진입 경로 | F1 메인 피드 상단 검색 아이콘 탭 |
| 연결 API  | B4 (필터/검색), B2 (추천순 정렬) |

---

### 화면 구성 (3개 상태)

#### 상태 1 — 검색 홈

| 요소               | 설명                                         |
| ------------------ | -------------------------------------------- |
| 검색창             | 상단 고정, 자동 포커스                       |
| 최근 검색어        | 최근 검색 키워드 목록 (X 버튼으로 개별 삭제) |
| 실시간 인기 검색어 | 순위 번호 + 키워드 목록 (B4에서 제공)        |
| 필터 설정 버튼     | "상세 필터" 또는 필터 아이콘 → 상태 2로 전환 |

#### 상태 2 — 필터 설정

| 필터 항목 | 선택 옵션                                        | 선택 방식 |
| --------- | ------------------------------------------------ | --------- |
| 지역      | 수도권 / 부산 / 강원 / 제주 / 전라 / 경상 / 충청 | 복수 선택 |
| 테마      | 맛집 / 명소 / 루트                               | 복수 선택 |
| 여행 유형 | 혼자 / 커플 / 가족 / 친구                        | 복수 선택 |

| 하단 버튼 | 동작                        |
| --------- | --------------------------- |
| 초기화    | 모든 필터 조건 초기화       |
| 결과 보기 | 선택 조건으로 검색 → 상태 3 |

#### 상태 3 — 필터 결과

| 요소           | 설명                                           |
| -------------- | ---------------------------------------------- |
| 활성 필터 태그 | 적용된 필터 칩 표시, X 버튼으로 개별 제거      |
| 결과 수        | "○개의 여행지를 찾았어요"                      |
| 정렬 드롭다운  | 추천순 / 인기순 / 최신순                       |
| 결과 리스트    | Masonry 2열 카드 (F1 카드와 동일 컴포넌트)     |
| 결과 없음      | "조건에 맞는 여행지가 없어요" + 필터 수정 유도 |

---

### 기능 요구사항

- 검색창에 텍스트 입력 시 실시간 자동완성 제안 **(구현 예정, 미확정)**
- 최근 검색어는 로컬 스토리지에 최대 10개 저장
- 실시간 인기 검색어는 B4 API에서 주기적으로 갱신 (5분 캐시)
- 필터 조건은 AND 연산으로 B4 API에 전달
- 활성 필터 칩에서 X 탭 시 해당 조건 제거 후 자동 재조회
- 정렬 변경 시 현재 필터 조건 유지한 채 재정렬

> **자동완성 기능 관련:** 검색어 자동완성은 B4 `/api/search/suggest` 엔드포인트로 구현 가능하나, 최종 프로젝트 일정에 따라 포함 여부를 확정할 예정입니다.

---

### 화면 플로우

```
검색 아이콘(F1)
    ↓
검색 홈(상태 1)
    ↓ 검색어 입력 + 엔터      ↓ 상세 필터 버튼
필터 결과(상태 3)          필터 설정(상태 2)
                                ↓ 결과 보기
                           필터 결과(상태 3)
                                ↓ 카드 탭
                           컨텐츠 상세(F2)
```

## B4 — 필터·검색 API

### 개요

| 항목      | 내용                                                |
| --------- | --------------------------------------------------- |
| API ID    | B4                                                  |
| API명     | 필터·검색 API                                       |
| 연결 화면 | F3 필터/검색, F6 온보딩 (기본값 저장), F7 테마 탐색 |

---

### 엔드포인트 목록

#### 1. 통합 검색

| 항목   | 내용                                    |
| ------ | --------------------------------------- |
| Method | GET                                     |
| URL    | `/api/search`                           |
| 설명   | 키워드 + 복합 필터 조건으로 콘텐츠 검색 |

**Query Parameters**

| 파라미터    | 타입   | 필수 | 설명                                               |
| ----------- | ------ | ---- | -------------------------------------------------- |
| keyword     | string | N    | 검색 키워드                                        |
| regions     | array  | N    | 지역 필터 (복수)                                   |
| themeIds    | array  | N    | 테마 필터 (복수)                                   |
| travelTypes | array  | N    | 여행 유형 필터 (복수)                              |
| sortBy      | string | N    | recommended / popular / latest (기본: recommended) |
| page        | number | N    | 페이지 번호 (기본 1)                               |
| limit       | number | N    | 페이지당 항목 수 (기본 20)                         |

**Response**

| 필드           | 타입    | 설명                                     |
| -------------- | ------- | ---------------------------------------- |
| contents       | array   | 콘텐츠 목록                              |
| totalCount     | number  | 전체 결과 수                             |
| hasNext        | boolean | 다음 페이지 존재 여부                    |
| appliedFilters | object  | 적용된 필터 정보 (프론트 필터 칩 표시용) |

---

#### 2. 실시간 인기 검색어

| 항목   | 내용                                     |
| ------ | ---------------------------------------- |
| Method | GET                                      |
| URL    | `/api/search/trending`                   |
| 설명   | 실시간 인기 검색어 순위 반환 (상위 10개) |

**Response**

| 필드     | 타입   | 설명                         |
| -------- | ------ | ---------------------------- |
| keywords | array  | 순위별 검색어 목록           |
| rank     | number | 순위                         |
| keyword  | string | 검색어                       |
| change   | string | 순위 변동 (up/down/same/new) |

---

#### 3. 자동완성 제안 (구현 예정)

| 항목   | 내용                                                                               |
| ------ | ---------------------------------------------------------------------------------- |
| Method | GET                                                                                |
| URL    | `/api/search/suggest`                                                              |
| 설명   | 입력 중인 키워드에 대한 자동완성 후보 반환 (최종 프로젝트에서 포함 여부 확정 예정) |

**Query Parameters**

| 파라미터 | 타입   | 필수 | 설명                        |
| -------- | ------ | ---- | --------------------------- |
| q        | string | Y    | 현재 입력 키워드 (2자 이상) |
| limit    | number | N    | 반환 개수 (기본 5)          |

---

#### 4. 필터 옵션 목록 조회

| 항목   | 내용                                                          |
| ------ | ------------------------------------------------------------- |
| Method | GET                                                           |
| URL    | `/api/search/filter-options`                                  |
| 설명   | 필터 설정 화면에 표시할 옵션 목록 반환 (지역, 테마, 여행유형) |

---

### 검색 처리 방식

| 항목             | 설명                                               |
| ---------------- | -------------------------------------------------- |
| 키워드 검색 대상 | 장소명, 해시태그, 설명, 지역명                     |
| 복합 필터 처리   | 동일 항목 내 복수 선택: OR / 항목 간: AND          |
| 검색어 통계      | 검색 요청 시 키워드 카운트 적재 → 인기 검색어 산정 |
| 갱신 주기        | 인기 검색어: 5분마다 집계                          |

**필터 예시**

```
지역: [부산, 강원] AND 테마: [맛집] AND 여행유형: [커플, 친구]
→ (지역=부산 OR 지역=강원) AND 테마=맛집 AND (여행유형=커플 OR 여행유형=친구)
```

---

### 비기능 요구사항

- 검색 응답 시간 목표: 1초 이내
- 키워드 검색은 형태소 분석 또는 LIKE 검색 적용
- 인기 검색어는 Redis 등 인메모리 캐시 활용 권장
- 자동완성 API는 입력 딜레이(debounce 300ms) 후 호출 권장
