# 기능 명세서 — 박예원

> 담당: F2 컨텐츠 상세 페이지 + B5 사용자 소장 데이터 API

---

## 서비스 소개

꼬잉꼬잉은 지역 기반 당일치기 여행지를 추천해주는 서비스입니다.
"극 P의 여행"을 컨셉으로, 사용자가 직접 찾지 않아도 관련 정보와 추천이 자연스럽게 제공되는 큐레이션 여행 서비스입니다.

## 담당 기능 선정 이유

F2 컨텐츠 상세 페이지는 꼬잉꼬잉의 핵심 철학인 "정보를 떠먹여주는 서비스"가 가장 잘 드러나는 화면입니다. 유저가 직접 찾지 않아도 관련 추천·지도·루트·해시태그가 자연스럽게 펼쳐지는 구조로, 서비스 전체 화면의 중심 허브 역할을 합니다. B5 소장 API는 사용자가 마음에 드는 여행지를 저장하고 관리하는 기능을 담당합니다.

## AI 대화 요약

꼬잉꼬잉 서비스 기획 방향 논의부터 시작해서,
전체 화면(F1~F8) 및 API(B1~B7) 구조를 도출하고,
담당 기능인 (본인 기능)의 화면 구성과 API 명세를
AI와의 대화를 통해 구체화하였습니다

## F2 — 컨텐츠 상세 페이지

### 개요

| 항목      | 내용                                                                              |
| --------- | --------------------------------------------------------------------------------- |
| 화면 ID   | F2                                                                                |
| 화면명    | 컨텐츠 상세 페이지                                                                |
| 진입 경로 | F1 메인 피드 / F3 필터 결과 / F7 테마 탐색 / F8 룰렛 결과 / F5 소장함에서 카드 탭 |
| 연결 API  | B1 (상세 데이터), B2 (추가 추천), B3 (루트 배너), B5 (찜 저장), B7 (해시태그)     |

---

### 화면 구성

> F2는 하나의 페이지를 위아래로 스크롤하는 구조입니다.
> 히어로 이미지부터 기본 정보까지 보이고, 스크롤을 내리면 추천·지도·CTA 영역이 이어집니다.

#### 히어로 이미지 영역

| 요소          | 설명                                                     |
| ------------- | -------------------------------------------------------- |
| 히어로 이미지 | 전체 너비 대형 이미지 (가로 스와이프로 복수 이미지 탐색) |
| 뒤로가기 버튼 | 이미지 위 좌상단 오버레이                                |
| 공유 버튼     | 이미지 위 우상단 오버레이                                |
| 찜(하트) 버튼 | 이미지 위 우상단 오버레이 (찜 상태 토글)                 |

#### 기본 정보 영역

| 요소      | 설명                                              |
| --------- | ------------------------------------------------- |
| 장소명    | 큰 텍스트로 강조                                  |
| 테마 태그 | 맛집 / 명소 / 루트 등 뱃지                        |
| 위치      | 지역명 + 상세 주소                                |
| 찜수      | 하트 아이콘 + 숫자                                |
| 여행 유형 | 혼자/커플/가족/친구 태그                          |
| 설명      | 장소 소개 텍스트 (더보기 버튼으로 축약/전체 토글) |

#### 정보 그리드 (2×2)

| 칸       | 내용                                   |
| -------- | -------------------------------------- |
| 영업시간 | 시간 표시 + 현재 영업 여부 (오픈/마감) |
| 가격대   | 알뜰(₩) / 보통(₩₩) / 여유(₩₩₩)         |
| 주차     | 가능 / 불가                            |
| 혼잡도   | 낮음 / 보통 / 높음 (색상 인디케이터)   |

#### 지도 미리보기 (스크롤 후 등장)

| 요소        | 설명                                                 |
| ----------- | ---------------------------------------------------- |
| 지도 썸네일 | 장소 위치 핀이 표시된 정적 지도 이미지               |
| 탭 동작     | 카카오맵 / 네이버지도 / 구글맵 앱 연동 (사용자 선택) |

#### 해시태그

| 요소          | 설명                                  |
| ------------- | ------------------------------------- |
| 해시태그 목록 | B7 태그 API에서 제공하는 태그 칩 목록 |
| 태그 탭       | 해당 태그 검색 결과(F3)로 이동        |

#### 추가 추천 카드 (가로 스크롤)

| 요소        | 설명                                    |
| ----------- | --------------------------------------- |
| 섹션 타이틀 | "이런 곳도 있어요"                      |
| 추천 카드   | B2 추천 API 기반, 소형 카드 가로 스크롤 |
| 카드 탭     | 해당 장소의 F2 상세 페이지로 이동       |

#### 같은 지역 루트 배너

| 요소    | 설명                                    |
| ------- | --------------------------------------- |
| 배너    | "○○ 지역 AI 추천 루트 보기" 형태        |
| 탭 동작 | B3 연결 → F4 AI 루트 결과 화면으로 이동 |

#### 하단 CTA 버튼 (화면 하단 고정)

| 버튼           | 동작                               |
| -------------- | ---------------------------------- |
| AI 루트 만들기 | → F4 AI 루트 결과 화면             |
| 소장하기       | → B5 찜 저장 + F5 소장함 이동 옵션 |

> **하단 CTA는 sticky(고정) 방식으로 구현합니다.** 스크롤 중에도 항상 화면 아래에 떠 있어, 사용자가 언제든 바로 누를 수 있습니다.

---

### 기능 요구사항

- 히어로 이미지 복수일 경우 스와이프 가능, 하단 인디케이터 표시
- 찜 버튼은 즉시 UI 반응 후 B5 API 비동기 호출 (낙관적 업데이트: 버튼을 누르는 순간 API 응답을 기다리지 않고 UI가 먼저 반응하는 방식)
- 비로그인 사용자 찜 탭 시 로그인 유도 팝업
- 지도 탭 시 앱이 없으면 웹 버전으로 폴백
- 설명 텍스트 기본 3줄 표시, "더보기"로 전체 펼침

---

### 화면 플로우

```
F1/F3/F7/F8/F5 카드 탭
    ↓
컨텐츠 상세(F2)
    ↓ AI 루트 만들기 (하단 고정 버튼)
F4 AI 루트 결과

    ↓ 소장하기 (하단 고정 버튼)
B5 찜 저장 → F5 소장함

    ↓ 추천 카드 탭 ("이런 곳도 있어요")
F2 컨텐츠 상세 (다른 장소)
```

## B5 — 사용자 소장 데이터 API

### 개요

| 항목      | 내용                       |
| --------- | -------------------------- |
| API ID    | B5                         |
| API명     | 사용자 소장(찜) 데이터 API |
| 연결 화면 | F2 컨텐츠 상세, F5 소장함  |
| 인증      | 전 엔드포인트 로그인 필요  |

---

### 엔드포인트 목록

#### 1. 소장함 목록 조회

| 항목   | 내용                         |
| ------ | ---------------------------- |
| Method | GET                          |
| URL    | `/api/users/me/collections`  |
| 설명   | 로그인 사용자의 찜 목록 반환 |

**Query Parameters**

| 파라미터 | 타입   | 필수 | 설명                             |
| -------- | ------ | ---- | -------------------------------- |
| type     | string | N    | `content` / `route` (기본: 전체) |
| page     | number | N    | 페이지 번호                      |
| limit    | number | N    | 페이지당 항목 수 (기본 20)       |

**Response**

| 필드        | 타입   | 설명           |
| ----------- | ------ | -------------- |
| collections | array  | 소장 항목 목록 |
| totalCount  | number | 전체 소장 수   |

---

#### 2. 콘텐츠 찜 추가

| 항목   | 내용                                               |
| ------ | -------------------------------------------------- |
| Method | POST                                               |
| URL    | `/api/users/me/collections/contents/{contentId}`   |
| 설명   | 특정 콘텐츠를 찜 목록에 추가, B1 likeCount +1 처리 |

**Response**

| 필드         | 타입   | 설명                |
| ------------ | ------ | ------------------- |
| collectionId | string | 생성된 소장 항목 ID |
| likeCount    | number | 업데이트된 찜수     |

---

#### 3. 콘텐츠 찜 취소

| 항목   | 내용                                             |
| ------ | ------------------------------------------------ |
| Method | DELETE                                           |
| URL    | `/api/users/me/collections/contents/{contentId}` |
| 설명   | 찜 목록에서 제거, B1 likeCount -1 처리           |

---

#### 4. 루트 소장 추가

| 항목   | 내용                                         |
| ------ | -------------------------------------------- |
| Method | POST                                         |
| URL    | `/api/users/me/collections/routes/{routeId}` |
| 설명   | AI 루트를 소장함에 저장 (B3 routeId 참조)    |

---

#### 5. 루트 소장 삭제

| 항목   | 내용                                         |
| ------ | -------------------------------------------- |
| Method | DELETE                                       |
| URL    | `/api/users/me/collections/routes/{routeId}` |
| 설명   | 소장한 루트 삭제                             |

---

#### 6. 찜 여부 확인

| 항목   | 내용                                                                |
| ------ | ------------------------------------------------------------------- |
| Method | GET                                                                 |
| URL    | `/api/users/me/collections/contents/{contentId}/status`             |
| 설명   | 특정 콘텐츠의 찜 여부 반환 (상세 페이지 진입 시 하트 상태 초기화용) |

**Response**

| 필드        | 타입    | 설명    |
| ----------- | ------- | ------- |
| isCollected | boolean | 찜 여부 |

---

### 데이터 모델 (소장 테이블)

| 컬럼         | 타입        | 설명                   |
| ------------ | ----------- | ---------------------- |
| collectionId | string (PK) | 소장 항목 ID           |
| userId       | string (FK) | 사용자 ID              |
| itemType     | string      | `content` 또는 `route` |
| itemId       | string      | 콘텐츠 또는 루트 ID    |
| createdAt    | datetime    | 찜 등록 시각           |

---

### 비기능 요구사항

- 찜 추가/취소 API 응답 시간 목표: 500ms 이내
- 동일 콘텐츠 중복 찜 방지 (409 Conflict 반환)
- likeCount 동시성 문제 방지를 위해 원자적 증감(atomic increment) 처리
