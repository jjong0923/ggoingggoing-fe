# 기능 명세서 — 박기범

> 담당: F7 테마 탐색 페이지 + B7 태그·카테고리 관리 API

---

## 서비스 소개

꼬잉꼬잉은 지역 기반 당일치기 여행지를 추천해주는 서비스입니다.
"극 P의 여행"을 컨셉으로, 사용자가 직접 찾지 않아도 관련 정보와 추천이 자연스럽게 제공되는 큐레이션 여행 서비스입니다.

## 담당 기능 선정 이유

F7 테마 탐색은 사용자가 "맛집", "명소", "루트" 중 원하는 테마를 골라 세부 카테고리까지 드릴다운 탐색할 수 있는 화면입니다. 단순 검색이 아닌 테마 중심의 발견 경험을 제공하는 역할을 합니다. B7 태그·카테고리 관리 API는 이 테마 구조 전체를 데이터로 뒷받침하며, F2 컨텐츠 상세의 해시태그 시스템과도 공유됩니다.

## AI 대화 요약

꼬잉꼬잉 서비스 기획 방향 논의부터 시작해서,
전체 화면(F1~F8) 및 API(B1~B7) 구조를 도출하고,
담당 기능인 (본인 기능)의 화면 구성과 API 명세를
AI와의 대화를 통해 구체화하였습니다

## F7 — 테마 탐색 페이지

### 개요

| 항목      | 내용                                          |
| --------- | --------------------------------------------- |
| 화면 ID   | F7                                            |
| 화면명    | 테마 탐색 페이지                              |
| 진입 경로 | 하단 탭바 "탐색" 탭                           |
| 연결 API  | B7 (테마/서브테마 목록, 콘텐츠 수), B4 (필터) |

---

### 화면 구성 (단계별 드릴다운)

> 아래 3단계는 **같은 탐색 흐름 안에서 순서대로 전환**되는 화면입니다.
> 1단계(테마 홈)에서 카드를 탭하면 2단계(서브테마 선택)로 이동하고,
> 서브테마를 선택하면 3단계(테마별 피드)로 넘어가는 구조입니다.

#### 1단계 — 테마 홈

| 요소      | 설명                                        |
| --------- | ------------------------------------------- |
| 테마 카드 | 맛집 / 명소 / 루트 3개 카드                 |
| 카드 구성 | 테마명 + 콘텐츠 수 + 미리보기 썸네일 이미지 |
| UI 형태   | 세로 스크롤 대형 카드 또는 그리드           |

#### 2단계 — 서브테마 선택

| 요소               | 설명                                                 |
| ------------------ | ---------------------------------------------------- |
| 서브테마 그리드    | 선택한 테마에 따라 세부 카테고리 표시                |
| 맛집 서브테마 예시 | 고기/구이, 면/국수, 해산물, 카페/디저트, 분식 등     |
| 명소 서브테마 예시 | 자연/힐링, 역사, 데이트, 가족, 계곡, 등산, 물놀이 등 |
| 지역 필터          | 서브테마 선택 후 지역 추가 필터 적용 가능            |
| 선택 방식          | 서브테마 복수 선택 가능                              |

#### 3단계 — 테마별 피드

| 요소           | 설명                                                           |
| -------------- | -------------------------------------------------------------- |
| 레이아웃       | Masonry 2열 카드 (메인 피드 F1과 동일 컴포넌트 재사용)         |
| 상단 필터 표시 | 현재 선택된 테마 + 서브테마 + 지역 칩 표시, X로 개별 제거 가능 |
| 결과 수 표시   | "○개의 여행지"                                                 |
| 정렬           | 추천순 / 인기순 / 최신순                                       |

---

### 기능 요구사항

- 테마 홈 → 서브테마 선택 → 테마별 피드 순서로 드릴다운 탐색
- 각 테마 카드의 콘텐츠 수는 B7 API에서 실시간 조회
- 서브테마 선택과 지역 필터는 AND 조건으로 B4 API에 전달
- 상단 필터 칩에서 X 버튼으로 개별 조건 제거 가능
- 서브테마 미선택 시 해당 테마 전체 콘텐츠 표시

---

### 화면 플로우

```
하단 탭 "탐색"
    ↓
1단계: 테마 홈 (맛집 / 명소 / 루트 카드)
    ↓ 카드 탭 → 다음 화면으로 전환
2단계: 서브테마 선택 + 지역 필터
    ↓ 서브테마 선택 후 결과 보기
3단계: 테마별 피드 (Masonry 카드 목록)
    ↓ 카드 탭
컨텐츠 상세(F2)
```

## B7 — 컨텐츠 태그·카테고리 관리 API

### 개요

| 항목      | 내용                                    |
| --------- | --------------------------------------- |
| API ID    | B7                                      |
| API명     | 태그·카테고리 관리 API                  |
| 연결 화면 | F7 테마 탐색, F2 컨텐츠 상세 (해시태그) |

---

### 엔드포인트 목록

#### 1. 테마 목록 조회

| 항목   | 내용                                               |
| ------ | -------------------------------------------------- |
| Method | GET                                                |
| URL    | `/api/categories/themes`                           |
| 설명   | 메인 테마(맛집/명소/루트) 목록과 각 콘텐츠 수 반환 |

**Response**

| 필드         | 타입   | 설명                            |
| ------------ | ------ | ------------------------------- |
| themeId      | string | 테마 고유 ID                    |
| themeName    | string | 테마명 (맛집, 명소, 루트)       |
| contentCount | number | 해당 테마 콘텐츠 수             |
| thumbnails   | array  | 미리보기 썸네일 이미지 URL 목록 |

---

#### 2. 서브테마 목록 조회

| 항목   | 내용                                         |
| ------ | -------------------------------------------- |
| Method | GET                                          |
| URL    | `/api/categories/themes/{themeId}/subtopics` |
| 설명   | 특정 테마의 서브테마 목록과 콘텐츠 수 반환   |

**Path Parameter**

| 파라미터 | 타입   | 설명         |
| -------- | ------ | ------------ |
| themeId  | string | 메인 테마 ID |

**Response**

| 필드         | 타입   | 설명                    |
| ------------ | ------ | ----------------------- |
| subtopicId   | string | 서브테마 ID             |
| subtopicName | string | 서브테마명              |
| contentCount | number | 해당 서브테마 콘텐츠 수 |

---

#### 3. 태그(해시태그) 목록 조회

| 항목   | 내용                                 |
| ------ | ------------------------------------ |
| Method | GET                                  |
| URL    | `/api/tags`                          |
| 설명   | 전체 해시태그 목록 및 사용 빈도 반환 |

**Query Parameter**

| 파라미터  | 타입   | 필수 | 설명                      |
| --------- | ------ | ---- | ------------------------- |
| contentId | string | N    | 특정 콘텐츠의 태그만 조회 |
| limit     | number | N    | 반환 개수 제한 (기본 20)  |

---

#### 4. 콘텐츠에 태그 연결

| 항목   | 내용                               |
| ------ | ---------------------------------- |
| Method | POST                               |
| URL    | `/api/tags/content/{contentId}`    |
| 설명   | 특정 콘텐츠에 태그 추가 (관리자용) |

---

#### 5. 카테고리 콘텐츠 수 일괄 갱신

| 항목   | 내용                                                       |
| ------ | ---------------------------------------------------------- |
| Method | PUT                                                        |
| URL    | `/api/categories/refresh-counts`                           |
| 설명   | 콘텐츠 추가/삭제 시 카테고리별 콘텐츠 수 재집계 (관리자용) |

---

### 데이터 구조

#### 테마 계층 구조 예시

```
맛집
├── 고기/구이
├── 면/국수
├── 해산물
├── 카페/디저트
└── 분식

명소
├── 자연/힐링
├── 역사
├── 데이트
├── 가족 여행
├── 계곡
├── 등산
└── 물놀이

루트
├── 당일치기 코스
└── 지역별 추천 루트
```

---

### 비기능 요구사항

- 테마/서브테마 목록은 캐싱 처리 (변경 빈도 낮음)
- 콘텐츠 수는 매일 1회 재집계 또는 콘텐츠 변경 시 트리거
- F2 컨텐츠 상세 해시태그와 F7 테마 탐색의 태그 시스템은 동일 테이블 공유
