# 기능 명세서 — 정석영

> 담당: F6 온보딩/스플래시 화면 + B6 사용자 인증 API

---

## 서비스 소개

꼬잉꼬잉은 지역 기반 당일치기 여행지를 추천해주는 서비스입니다.
"극 P의 여행"을 컨셉으로, 사용자가 직접 찾지 않아도 관련 정보와 추천이 자연스럽게 제공되는 큐레이션 여행 서비스입니다.

## 담당 기능 선정 이유

F6 온보딩은 사용자가 꼬잉꼬잉을 처음 만나는 화면입니다. 여기서 수집한 취향 데이터(테마·지역·여행 스타일)가 이후 메인 피드 추천과 필터 기본값으로 연결되기 때문에, 서비스 전체 개인화의 시작점 역할을 합니다. B6 인증 API는 이 온보딩 흐름과 직결되어 로그인/비로그인 분기를 처리합니다.

## AI 대화 요약

꼬잉꼬잉 서비스 기획 방향 논의부터 시작해서,
전체 화면(F1~F8) 및 API(B1~B7) 구조를 도출하고,
담당 기능인 (본인 기능)의 화면 구성과 API 명세를
AI와의 대화를 통해 구체화하였습니다

## F6 — 온보딩 / 스플래시 화면

### 개요

| 항목      | 내용                                                   |
| --------- | ------------------------------------------------------ |
| 화면 ID   | F6                                                     |
| 화면명    | 온보딩 / 스플래시                                      |
| 진입 시점 | 앱 최초 실행 또는 비로그인 상태                        |
| 연결 API  | B6 (회원가입/로그인), B2 (취향 저장), B4 (필터 기본값) |

---

### 화면 구성 (단계별)

#### 1단계 — 스플래시

| 요소                    | 설명                                            |
| ----------------------- | ----------------------------------------------- |
| 로고                    | 꼬잉꼬잉 서비스 로고 중앙 표시                  |
| 시작하기 버튼           | 회원가입/로그인 플로우 진입                     |
| 둘러보기(건너뛰기) 버튼 | 취향 설정 없이 게스트 모드로 메인 피드(F1) 이동 |

#### 2단계 — 테마 선택

| 요소        | 설명                                          |
| ----------- | --------------------------------------------- |
| 선택 항목   | 맛집 / 명소 / 힐링 / 등산 / 물놀이 / 지역축제 |
| 선택 방식   | 복수 선택 가능                                |
| UI 형태     | 아이콘 + 텍스트 카드 그리드                   |
| 선택 최소값 | 1개 이상 선택 시 다음 단계 활성화             |

#### 3단계 — 지역 선택

| 요소      | 설명                                                |
| --------- | --------------------------------------------------- |
| 선택 항목 | 수도권 / 부산 / 강원 / 제주 / 전라 / 경상 / 충청 등 |
| 선택 방식 | 복수 선택 가능                                      |
| UI 형태   | 텍스트 칩 또는 지도 기반 선택 UI                    |

#### 4단계 — 여행 스타일 선택

| 항목      | 옵션                      |
| --------- | ------------------------- |
| 이동시간  | 당일치기 / 1박2일         |
| 동행 유형 | 혼자 / 커플 / 가족 / 친구 |
| 예산      | 알뜰 / 보통 / 여유        |

---

### 기능 요구사항

- 모든 단계에서 이전 단계로 돌아갈 수 있는 Back 버튼 제공
- 건너뛰기 선택 시 기본값(전체 지역, 전체 테마)으로 B2 추천 알고리즘에 전달
- 온보딩 완료 데이터는 B2(추천 기본값), B4(필터 기본값)으로 전달
- 온보딩 완료 후 자동으로 메인 피드(F1)로 이동
- 재방문 사용자(로그인 상태)는 온보딩 스킵, 바로 F1 진입

---

### 화면 플로우

```
스플래시(1단계)
    ↓ 시작하기                    ↓ 건너뛰기
테마 선택(2단계)          게스트 모드로 메인 피드(F1) 이동
    ↓                     (취향 데이터 없이 인기순 피드 노출,
지역 선택(3단계)           찜·루트 저장 등 데이터 저장 기능은 이용 불가)
    ↓
여행 스타일(4단계)
    ↓
→ B6 로그인/가입 처리
→ B2 취향 저장
→ 메인 피드(F1)
```

> **게스트 모드란?** 로그인 없이 서비스를 둘러볼 수 있는 상태입니다. 피드 조회·검색 등 탐색 기능은 이용 가능하지만, 찜 저장·AI 루트 저장 등 사용자 데이터를 기록하는 기능은 로그인 후 이용 가능합니다.

## B6 — 사용자 인증 API

### 개요

| 항목      | 내용            |
| --------- | --------------- |
| API ID    | B6              |
| API명     | 사용자 인증 API |
| 연결 화면 | F6 온보딩       |

---

### 엔드포인트 목록

#### 1. 회원가입

| 항목   | 내용                 |
| ------ | -------------------- |
| Method | POST                 |
| URL    | `/api/auth/signup`   |
| 설명   | 신규 사용자 회원가입 |

**Request Body**

| 파라미터 | 타입   | 필수 | 설명                |
| -------- | ------ | ---- | ------------------- |
| email    | string | Y    | 이메일 주소         |
| password | string | Y    | 비밀번호 (8자 이상) |
| nickname | string | Y    | 닉네임              |

**Response**

| 필드        | 타입   | 설명             |
| ----------- | ------ | ---------------- |
| userId      | string | 생성된 사용자 ID |
| accessToken | string | 인증 토큰        |
| message     | string | 처리 결과 메시지 |

---

#### 2. 로그인

| 항목   | 내용                   |
| ------ | ---------------------- |
| Method | POST                   |
| URL    | `/api/auth/login`      |
| 설명   | 이메일/비밀번호 로그인 |

**Request Body**

| 파라미터 | 타입   | 필수 | 설명        |
| -------- | ------ | ---- | ----------- |
| email    | string | Y    | 이메일 주소 |
| password | string | Y    | 비밀번호    |

**Response**

| 필드        | 타입    | 설명                                  |
| ----------- | ------- | ------------------------------------- |
| userId      | string  | 사용자 ID                             |
| accessToken | string  | 인증 토큰                             |
| isNewUser   | boolean | 신규 사용자 여부 (온보딩 진행 판단용) |

---

#### 3. 소셜 로그인 (선택)

| 항목   | 내용                      |
| ------ | ------------------------- |
| Method | POST                      |
| URL    | `/api/auth/social`        |
| 설명   | 카카오 / 구글 소셜 로그인 |

**Request Body**

| 파라미터    | 타입   | 필수 | 설명                  |
| ----------- | ------ | ---- | --------------------- |
| provider    | string | Y    | `kakao` 또는 `google` |
| accessToken | string | Y    | 소셜 플랫폼 토큰      |

---

#### 4. 토큰 갱신

| 항목   | 내용                    |
| ------ | ----------------------- |
| Method | POST                    |
| URL    | `/api/auth/refresh`     |
| 설명   | 만료된 accessToken 갱신 |

---

#### 5. 로그아웃

| 항목   | 내용                     |
| ------ | ------------------------ |
| Method | POST                     |
| URL    | `/api/auth/logout`       |
| 설명   | 세션 종료 및 토큰 무효화 |

---

### 비기능 요구사항

- 비로그인(게스트) 상태에서도 서비스 조회 기능은 이용 가능
- 게스트는 찜(B5), AI 루트 저장 등 사용자 데이터 저장 기능 이용 불가
- 비밀번호는 bcrypt 등으로 해시 저장
- accessToken 만료 시간: 1시간 / refreshToken: 30일
