# 기능 명세서 — 김주영

> 담당: F5 소장함(찜) 페이지 + F8 랜덤 룰렛 화면 + B2 추천 알고리즘 API

---

## 서비스 소개

꼬잉꼬잉은 지역 기반 당일치기 여행지를 추천해주는 서비스입니다.
"극 P의 여행"을 컨셉으로, 사용자가 직접 찾지 않아도 관련 정보와 추천이 자연스럽게 제공되는 큐레이션 여행 서비스입니다.

## 담당 기능 선정 이유

F5 소장함은 사용자가 마음에 든 여행지를 모아두고 폴더별로 관리하는 개인 공간입니다. F8 랜덤 룰렛은 꼬잉꼬잉의 "극 P 여행" 컨셉을 가장 직접적으로 표현하는 기능으로, 고민 없이 여행지를 뽑아주는 핵심 차별점입니다. B2 추천 알고리즘 API는 메인 피드 순서, 상세 페이지 추천, 룰렛 후보 목록 등 서비스 전반의 개인화를 담당하는 두뇌 역할입니다.

## AI 대화 요약

꼬잉꼬잉 서비스 기획 방향 논의부터 시작해서,
전체 화면(F1~F8) 및 API(B1~B7) 구조를 도출하고,
담당 기능인 (본인 기능)의 화면 구성과 API 명세를
AI와의 대화를 통해 구체화하였습니다

## F5 — 소장함(찜) 페이지

### 개요

사용자가 찜한 컨텐츠를 한곳에서 모아보고 폴더별로 관리하는 페이지입니다.

### 화면 구성

| 영역         | 내용                                                                                                     |
| ------------ | -------------------------------------------------------------------------------------------------------- |
| 상단 헤더    | "소장함" 타이틀 + 검색 아이콘 + 더보기(편집 모드 진입) 버튼                                              |
| 탭 바        | 폴더 / 전체 / 루트 탭 전환                                                                               |
| 폴더 탭      | 사용자가 만든 폴더를 2열 그리드로 표시. 각 폴더는 안에 담긴 이미지 4개를 2×2로 자동 조합하여 썸네일 생성 |
| 전체 탭      | Masonry 2열 카드 레이아웃으로 전체 소장 목록 표시                                                        |
| 루트 탭      | 저장한 AI 루트 목록 표시. 없을 경우 빈 상태(Empty State) UI + 루트 만들기 유도                           |
| 새 폴더 버튼 | 폴더 탭에서 새 폴더 생성 가능                                                                            |

### 폴더 상세 화면

- 폴더 탭에서 폴더 카드 탭 시 해당 폴더 내 항목만 Masonry로 표시
- 정렬: 최근 저장순 / 지역순 / 테마순
- 카드 우상단 하트 버튼으로 소장 해제 (B5 DELETE 호출)
- 상단 편집 버튼으로 폴더명 변경 · 폴더 삭제 가능

### 기능 목록

| 기능                | 설명                                    |
| ------------------- | --------------------------------------- |
| 소장 목록 조회      | 로그인 사용자의 찜 목록 불러오기        |
| 폴더 생성/삭제/수정 | 사용자 정의 폴더로 소장 항목 분류       |
| 카테고리 필터       | 테마별(맛집/명소/루트) 필터링           |
| 찜 해제             | 카드 내 하트 버튼으로 개별 해제         |
| 일괄 삭제           | 편집 모드 진입 후 복수 선택 삭제        |
| 상세 이동           | 카드 탭 시 F2 컨텐츠 상세 페이지로 이동 |
| 빈 상태 처리        | 소장 항목 없을 때 탐색 유도 UI 표시     |

### 연결 API

- **B5** — 찜 목록 조회 / 삭제 (주 담당자: 박예원)
- B5 API 명세 연동 필요: `GET /collections`, `DELETE /collections/{id}`

## F8 — 랜덤 룰렛 화면

### 개요

조건을 설정하거나 설정 없이도 랜덤으로 당일치기 여행지를 뽑아주는 화면입니다. "극 P 유저"의 핵심 기능으로, 고민 없이 바로 떠날 수 있도록 설계되었습니다.

### 화면 구성 (3단계)

#### 1단계 — 조건 설정

| 항목      | 내용                                                           |
| --------- | -------------------------------------------------------------- |
| 지역 선택 | 수도권 / 부산 / 강원 / 제주 / 전라 등 (선택 안 해도 진행 가능) |
| 테마 선택 | 맛집 / 명소 / 루트 (복수 선택 가능)                            |
| 여행 유형 | 혼자 / 커플 / 가족 / 친구                                      |
| 하단 버튼 | "룰렛 돌리기!" CTA                                             |

> 조건을 하나도 선택하지 않아도 룰렛 실행 가능 (전체 후보 대상, 진정한 랜덤)

#### 2단계 — 룰렛 대기 & 애니메이션

| 항목            | 내용                                      |
| --------------- | ----------------------------------------- |
| 룰렛 판         | 조건 기반 후보 여행지 이름이 섹터에 배치  |
| 회전 애니메이션 | 빠르게 돌다가 서서히 감속하며 멈추는 효과 |
| 후보 수         | B2 API에서 최대 8개 후보 반환 권장        |
| 로딩 처리       | 후보 데이터 로딩 중 스피너 표시           |

#### 3단계 — 결과 카드

| 항목        | 내용                                     |
| ----------- | ---------------------------------------- |
| 당첨 여행지 | 장소명 + 대표 이미지 + 위치 + 한줄 설명  |
| CTA 버튼 1  | "AI 루트 만들기" → F4 이동               |
| CTA 버튼 2  | "소장하기" → B5 찜 저장 + F5 연결        |
| CTA 버튼 3  | "다시 돌리기" → 1단계로 리셋 (조건 유지) |

### 기능 목록

| 기능            | 설명                                            |
| --------------- | ----------------------------------------------- |
| 조건 설정       | 지역·테마·여행유형 선택 (선택 없이도 실행 가능) |
| 후보 목록 요청  | B2 API에 조건 전달 → 후보 여행지 리스트 수신    |
| 룰렛 애니메이션 | 후보 이름을 섹터에 배치 후 회전 → 감속 → 정지   |
| 결과 표시       | 당첨 여행지 상세 카드 노출                      |
| 소장 연동       | 결과 화면에서 바로 찜 저장 가능                 |
| 루트 연동       | 결과 화면에서 F4 AI 루트로 이동                 |
| 다시 돌리기     | 직전 조건 유지하며 재실행                       |

### 연결 API

- **B2** — 조건 기반 후보 목록 추출 (자체 담당)
- **B5** — 결과 화면에서 소장 저장 (박예원 담당 연동)

## B2 — 추천 알고리즘 API

### 개요

온보딩 취향 데이터 및 사용자 행동 기반으로 콘텐츠 추천 순서를 결정하고, 룰렛 후보 목록을 제공하는 API입니다. 메인 피드·컨텐츠 상세·룰렛 등 서비스 전반에서 호출됩니다.

### 엔드포인트 목록

#### 1. 메인 피드 추천 순서

```
GET /recommendations/feed
```

| 파라미터 | 타입              | 설명                                      |
| -------- | ----------------- | ----------------------------------------- |
| userId   | string (optional) | 로그인 사용자 ID (없으면 인기순 fallback) |
| region   | string (optional) | 지역 필터                                 |
| theme    | string (optional) | 테마 필터                                 |
| page     | int               | 페이지 번호                               |
| limit    | int               | 페이지당 항목 수 (기본 20)                |

**응답 예시**

```json
{
  "contents": [
    {
      "id": "content_001",
      "name": "성심당",
      "region": "대전",
      "theme": "맛집",
      "isHot": true,
      "score": 0.95
    }
  ],
  "totalCount": 120,
  "page": 1
}
```

---

#### 2. 룰렛 후보 목록 추출

```
GET /recommendations/roulette
```

| 파라미터   | 타입                | 설명                                   |
| ---------- | ------------------- | -------------------------------------- |
| region     | string (optional)   | 지역 조건                              |
| theme      | string[] (optional) | 테마 조건 (복수)                       |
| travelType | string (optional)   | 여행 유형 (solo/couple/family/friends) |
| count      | int                 | 후보 수 (기본 8, 최대 8)               |

**응답 예시**

```json
{
  "candidates": [
    {
      "id": "content_042",
      "name": "속초 물회",
      "region": "강원",
      "theme": "맛집",
      "description": "동해바다 앞 신선한 물회 한 그릇",
      "imageUrl": "https://..."
    }
  ]
}
```

---

#### 3. 컨텐츠 상세 "이런 곳도 있어요" 추천

```
GET /recommendations/related/{contentId}
```

| 파라미터  | 타입   | 설명                     |
| --------- | ------ | ------------------------ |
| contentId | string | 현재 보고 있는 컨텐츠 ID |
| limit     | int    | 추천 수 (기본 6)         |

**응답 예시**

```json
{
  "related": [
    {
      "id": "content_077",
      "name": "뭉티기",
      "region": "대구",
      "theme": "맛집",
      "imageUrl": "https://..."
    }
  ]
}
```

---

#### 4. 온보딩 취향 저장

```
POST /recommendations/preference
```

| 필드        | 타입     | 설명                    |
| ----------- | -------- | ----------------------- |
| userId      | string   | 사용자 ID               |
| themes      | string[] | 선택한 테마             |
| regions     | string[] | 선택한 지역             |
| travelStyle | object   | 이동시간/동행/예산 정보 |

---

### 추천 알고리즘 로직 (기본 방향)

| 단계 | 내용                                                 |
| ---- | ---------------------------------------------------- |
| 1    | 온보딩 취향 데이터(테마·지역·여행스타일) 가중치 적용 |
| 2    | 컨텐츠 조회수·찜수 기반 인기 스코어 반영             |
| 3    | 사용자가 이미 본 컨텐츠 하단 배치 (중복 노출 최소화) |
| 4    | 비로그인(게스트) 시 인기순 fallback                  |
| 5    | 룰렛 후보: 조건 매칭 컨텐츠 중 랜덤 샘플링           |

### 연결 화면

- F1 메인 피드 (한종민 담당)
- F2 컨텐츠 상세 — "이런 곳도 있어요" (박예원 담당)
- F6 온보딩 — 취향 저장 (정석영 담당)
- F8 랜덤 룰렛 — 후보 목록 (자체)
