# 기능 명세서 — 한종민

> 담당: F1 메인 피드 + B1 컨텐츠 DB 설계 및 CRUD API

---

## 서비스 소개

꼬잉꼬잉은 지역 기반 당일치기 여행지를 추천해주는 서비스입니다.
"극 P의 여행"을 컨셉으로, 사용자가 직접 찾지 않아도 관련 정보와 추천이 자연스럽게 제공되는 큐레이션 여행 서비스입니다.

## 담당 기능 선정 이유

F1 메인 피드는 사용자가 앱에 들어왔을 때 가장 먼저 보는 화면으로, 꼬잉꼬잉의 첫인상을 결정합니다. 핀터레스트식 카드 레이아웃으로 여행지를 시각적으로 탐색할 수 있게 합니다. B1 컨텐츠 DB는 서비스의 모든 장소·맛집·명소 데이터를 저장하고 제공하는 핵심 데이터 레이어로, 피드를 포함한 대부분의 화면이 이 API를 기반으로 동작합니다.

## AI 대화 요약

꼬잉꼬잉 서비스 기획 방향 논의부터 시작해서,
전체 화면(F1~F8) 및 API(B1~B7) 구조를 도출하고,
담당 기능인 (본인 기능)의 화면 구성과 API 명세를
AI와의 대화를 통해 구체화하였습니다

---

## F1 — 메인 피드

### 개요

| 항목      | 내용                               |
| --------- | ---------------------------------- |
| 화면 ID   | F1                                 |
| 화면명    | 메인 피드                          |
| 진입 경로 | 온보딩 완료 후 / 하단 탭바 "홈"    |
| 연결 API  | B1 (콘텐츠 데이터), B2 (추천 순서) |

---

### 화면 구성

#### 상단 헤더

| 요소        | 설명                             |
| ----------- | -------------------------------- |
| 로고        | 꼬잉꼬잉 로고 (좌측)             |
| 검색 아이콘 | 탭 시 F3 필터/검색 화면으로 이동 |
| 알림 아이콘 | 알림 목록 진입                   |

#### 필터 칩 바

| 요소    | 설명                                    |
| ------- | --------------------------------------- |
| 칩 목록 | 전체 / 맛집 / 명소 / 루트 / 지역별      |
| 방식    | 가로 스크롤                             |
| 기본값  | "전체" 선택 상태                        |
| 선택 시 | 해당 카테고리로 피드 필터링 (B1 재요청) |

#### 메인 카드 피드

| 요소       | 설명                                                                             |
| ---------- | -------------------------------------------------------------------------------- |
| 레이아웃   | 핀터레스트식 Masonry 2열                                                         |
| 스크롤     | 무한 스크롤 (페이지네이션)                                                       |
| 카드 구성  | 이미지 + 장소명 + 지역 + 테마 태그 + 찜 버튼 + HOT 배지                          |
| HOT 배지   | 최근 7일 조회수 또는 전체 찜수 상위 10% 콘텐츠에 표시 (B1에서 isHot 플래그 제공) |
| 찜 버튼    | 하트 아이콘, 탭 시 B5 소장 API 호출                                              |
| 카드 탭 시 | F2 컨텐츠 상세 페이지로 이동                                                     |

#### 하단 탭바

| 탭   | 연결 화면           |
| ---- | ------------------- |
| 홈   | F1 메인 피드 (현재) |
| 탐색 | F7 테마 탐색        |
| 룰렛 | F8 랜덤 룰렛        |
| 소장 | F5 소장함           |
| MY   | 마이페이지          |

---

### 기능 요구사항

- 초기 로드 시 B2 추천 알고리즘 기반으로 피드 순서 결정
- 필터 칩 선택 시 해당 카테고리 콘텐츠만 노출
- 무한 스크롤: 스크롤 끝 도달 시 다음 페이지 자동 로드
- 로그인 상태: 찜 버튼 즉시 반응 + B5 API 호출
- 비로그인(게스트): 찜 버튼 탭 시 로그인 유도 팝업
- 이미지 지연 로딩(lazy load) 적용으로 초기 로드 성능 최적화

---

### 화면 플로우

```
메인 피드(F1)
    ↓ 카드 탭
컨텐츠 상세(F2)

    ↓ 검색 아이콘
필터/검색(F3)

    ↓ 필터 칩 선택
카테고리별 피드 갱신
```

## B1 — 컨텐츠 DB 설계 및 CRUD API

### 개요

| 항목      | 내용                         |
| --------- | ---------------------------- |
| API ID    | B1                           |
| API명     | 컨텐츠 DB 설계 및 CRUD API   |
| 연결 화면 | F1 메인 피드, F2 컨텐츠 상세 |

---

### 데이터 모델 (컨텐츠 테이블)

| 컬럼          | 타입        | 설명                        |
| ------------- | ----------- | --------------------------- |
| contentId     | string (PK) | 콘텐츠 고유 ID              |
| name          | string      | 장소명                      |
| region        | string      | 지역 (부산, 강원 등)        |
| address       | string      | 상세 주소                   |
| themeId       | string (FK) | 메인 테마 ID (B7 참조)      |
| subtopicIds   | array       | 서브테마 ID 목록            |
| tags          | array       | 해시태그 목록               |
| description   | string      | 장소 설명                   |
| imageUrls     | array       | 이미지 URL 목록             |
| businessHours | string      | 영업시간                    |
| priceRange    | string      | 가격대 (알뜰/보통/여유)     |
| parking       | boolean     | 주차 가능 여부              |
| congestion    | string      | 혼잡도 (낮음/보통/높음)     |
| latitude      | float       | 위도                        |
| longitude     | float       | 경도                        |
| viewCount     | number      | 조회수                      |
| likeCount     | number      | 찜수                        |
| isHot         | boolean     | HOT 배지 여부               |
| travelType    | string      | 여행 유형 (당일치기/1박2일) |
| createdAt     | datetime    | 생성일                      |
| updatedAt     | datetime    | 수정일                      |

---

### 엔드포인트 목록

#### 1. 콘텐츠 목록 조회 (피드)

| 항목   | 내용                         |
| ------ | ---------------------------- |
| Method | GET                          |
| URL    | `/api/contents`              |
| 설명   | 메인 피드용 콘텐츠 목록 반환 |

**Query Parameters**

| 파라미터 | 타입   | 필수 | 설명                                   |
| -------- | ------ | ---- | -------------------------------------- |
| themeId  | string | N    | 테마 필터                              |
| region   | string | N    | 지역 필터                              |
| page     | number | N    | 페이지 번호 (기본 1)                   |
| limit    | number | N    | 페이지당 항목 수 (기본 20)             |
| sortBy   | string | N    | 정렬 기준 (recommended/popular/latest) |

**Response**

| 필드       | 타입    | 설명                  |
| ---------- | ------- | --------------------- |
| contents   | array   | 콘텐츠 목록           |
| totalCount | number  | 전체 콘텐츠 수        |
| hasNext    | boolean | 다음 페이지 존재 여부 |

---

#### 2. 콘텐츠 상세 조회

| 항목   | 내용                                       |
| ------ | ------------------------------------------ |
| Method | GET                                        |
| URL    | `/api/contents/{contentId}`                |
| 설명   | 특정 콘텐츠 상세 정보 반환, 조회수 +1 처리 |

---

#### 3. 콘텐츠 등록 (관리자)

| 항목   | 내용                             |
| ------ | -------------------------------- |
| Method | POST                             |
| URL    | `/api/contents`                  |
| 설명   | 신규 콘텐츠(장소/맛집/명소) 등록 |

---

#### 4. 콘텐츠 수정 (관리자)

| 항목   | 내용                        |
| ------ | --------------------------- |
| Method | PUT                         |
| URL    | `/api/contents/{contentId}` |
| 설명   | 기존 콘텐츠 정보 수정       |

---

#### 5. 콘텐츠 삭제 (관리자)

| 항목   | 내용                           |
| ------ | ------------------------------ |
| Method | DELETE                         |
| URL    | `/api/contents/{contentId}`    |
| 설명   | 콘텐츠 삭제 (소프트 삭제 권장) |

---

### HOT 배지 기준

| 조건         | 설명                              |
| ------------ | --------------------------------- |
| 기준 1       | 최근 7일 조회수 상위 10%          |
| 기준 2       | 전체 찜수 상위 10%                |
| isHot 플래그 | 위 조건 중 하나 이상 충족 시 true |
| 갱신 주기    | 매일 자정 배치 처리               |

---

### 비기능 요구사항

- 콘텐츠 목록 API는 페이지네이션 필수 적용
- 상세 조회 시 조회수 증가 처리 (비동기 처리 권장)
- 이미지는 CDN URL로 제공
- 검색/필터 성능을 위해 region, themeId 컬럼 인덱스 적용
